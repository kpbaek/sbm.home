변경내역

마크는 리엑터(CodeIgniter Reactor)를 통해서 추가된 사항입니다.

버전 2.1.0

Release Date: November 14, 2011

  • 일반적 변화
    • 보안 라이브러리의 파라미터 인젝션(injection flaw) 취약점 수정.HTML5 취약점 보강을 위해 XSS 필터 강화.
    • 콜백 검증규칙(validation rule)에 파라미터를 넘겨줄 수 있음.
    • 일반함수(Common functions)에 html_escape() 추가하여 XSS에 대한 보안강화.
  • 헬퍼
    • 문자열헬퍼(String Helper)에 increment_string()를 추가함."foo" 문자열을 "foo-1","foo-1","foo-2" 따위로 가공할 수 있음..
    • 폼헬퍼가 변경됨- form_open_multipart에서 action지정이 옵션이됨. Fixes (#65)
    • url_title() 에서 추가 대쉬문자가 trim됨(처음과 끝부분에 있다면 제거).
    • 문자열헬퍼(String Helper)의random_string() 메서드 속도향상.
  • 데이터베이스
    • CUBRID 드라이버가 추가됨 . CUBRID 팀 고마우이.헐헐.
    • PDO 드라이버 추가됨.(역주:이하 나중에 번역하겠습니다.)
    • Typecast limit and offset in the Database driver to integers to avoid possible injection.
    • Added additional option 'none' for the optional third argument for $this->db->like() in the Database driver.
    • Added $this->db->insert_batch() support to the OCI8 (Oracle) driver.
  • Libraries
    • Changed $this->cart->insert() in the Cart library to return the Row ID if a single item was inserted successfully.
    • Added support to set an optional parameter in your callback rules of validation using the Form Validation library.
    • Added a Migration library to assist with applying incremental updates to your database schema.
    • Driver children can be located in any package path.
    • Added is_unique to the Form Validation library.
    • Added $config['use_page_numbers'] to the Pagination library, which enables real page numbers in the URI.
    • Added TLS and SSL Encryption for SMTP.
  • Core
    • Changed private functions in URI library to protected so MY_URI can override them.
    • Removed CI_CORE boolean constant from CodeIgniter.php (there are no longer different Reactor and Core versions).

버그수정( 2.1.0)

  • Fixed #378 Robots identified as regular browsers by the User Agent class.
  • If a config class was loaded first then a library with the same name is loaded, the config would be ignored.
  • Fixed a bug (Reactor #19) where 1) the 404_override route was being ignored in some cases, and 2) auto-loaded libraries were not available to the 404_override controller when a controller existed but the requested method did not.
  • Fixed a bug (Reactor #89) where MySQL export would fail if the table had hyphens or other non alphanumeric/underscore characters.
  • Fixed a bug (#200) where MySQL queries would be malformed after calling $this->db->count_all() then $this->db->get()
  • Fixed bug #105 that stopped query errors from being logged unless database debugging was enabled
  • Fixed a bug (#160) - Removed unneeded array copy in the file cache driver.
  • Fixed a bug (#150) - field_data() now correctly returns column length.
  • Fixed a bug (#8) - load_class() now looks for core classes in APPPATH first, allowing them to be replaced.
  • Fixed a bug (#24) - ODBC database driver called incorrect parent in __construct().
  • Fixed a bug (#85) - OCI8 (Oracle) database escape_str() function did not escape correct.
  • Fixed a bug (#344) - Using schema found in Saving Session Data to a Database, system would throw error "user_data does not have a default value" when deleting then creating a session.
  • Fixed a bug (#112) - OCI8 (Oracle) driver didn't pass the configured database character set when connecting.
  • Fixed a bug (#182) - OCI8 (Oracle) driver used to re-execute the statement whenever num_rows() is called.
  • Fixed a bug (#82) - WHERE clause field names in the DB update_string() method were not escaped, resulting in failed queries in some cases.
  • Fixed a bug (#89) - Fix a variable type mismatch in DB display_error() where an array is expected, but a string could be set instead.
  • Fixed a bug (#467) - Suppress warnings generated from get_magic_quotes_gpc() (deprecated in PHP 5.4)
  • Fixed a bug (#484) - First time _csrf_set_hash() is called, hash is never set to the cookie (in Security.php).
  • Fixed a bug (#60) - Added _file_mime_type() method to the File Uploading Library in order to fix a possible MIME-type injection (also fixes bug #394).
  • Fixed a bug (#537) - Support for all wav type in browser.
  • Fixed a bug (#576) - Using ini_get() function to detect if apc is enabled or not.
  • Fixed invalid date time format in Date helper and XMLRPC library.

버전 2.0.3

일자: 2011년 8월 20일

  • 보안
    • 데이터베이스 클라이언트에서 멀티바이트 문자셋(한글 같은 아시아권 언어)을 사용하는 경우 발생할 수 있는 SQL 인젝션 공격 방지 기능이 MySQL 과 MySQLi 드라이버에 추가되었습니다.

      PHP 5.2.3 상위버전(<)과 MySQL 5.0.7 하위버전(>) 에서 mysql_set_charset() 사용시 SQL 인젝션 공격에 취약하게 됩니다. Latin-1, UTF-8, ASCII 이하의 문자셋을 사용할때는 안전합니다.

      데이터베이스 연결시에 멀티바이트 문자셋을 사용할때는, SQL 인젝션 공격에 취약한지 여부를 꼭 확인하세요!

  • 일반
    • index.php 주석에 철자 틀린 것 고쳤음.
    • 세션클래스의 userdata가 출력 프로파일러(output profiler)에 추가되었습니다. 또한, HTTP헤더,세션데이터, 설정변수(Config Variables)를 보이거나 감추는 기능이 추가되었습니다.
    • EXT 상수를 더이상 내부적으로 사용하지 않습니다.
    • welcome_message 뷰 파일 및 기본 에러 템플릿이 없데이트 되었습니다.
    • PostgreSQL 데이터베이스 드라이버에 insert_batch() 함수가 추가되었습니다.
    • mimes.php에 "application/x-csv" 가 추가되었습니다..
    • 이메일 라이브러리에서 Email library 첨부파일 이름에 점(.)이 포함되면, 잘못된 MIME-타입을 사용하던 문제가 수정되었습니다..
  • 헬퍼(Helpers )
    • html헬퍼의 heading() 함수에 세번째 파라미터를 추가하였습니다. 세번째 파라미터는 html 속성(예를 들어 class 따위)를 추가하기위해 사용합니다.
  • 라이브러리(Libraries )
    • user_agent 문자열에 대해 더 길게 일치하도록 Session을 수정하였습니다. 이전버전에서 업데이트 하기 페이지를 참조하세요.
    • Form Validation library에 is_unique 가 추가되었습니다.
    • Database Driver 에 $this->db->set_dbprefix() 가 추가되었습니다.
    • Cart Library 의 $this->cart->insert() 가 수정되었습니다. 하나의 아이템이 성공적으로 삽입(insert)되었을경우 Row ID 를 리턴합니다.
    • Loader library 에$this->load->get_var() 가 추가되었습니다.$this->load->view() 와$this->load->vars().를 통해 설정된 전역변수(global vars)를 추출하기 위해 사용합니다.
    • escape_str() 대신 escape()함수를 사용하여 인용부호를 추가하도록$this->db->having() 가 수정되었습니다.

버그수정

  • ENVIRONMENT 가 예약된 상수로 추가되었습니다. (Reactor #196)
  • 서버를 체크하는 로직이 SCRIPT_NAME 가 정의되었는지 확인하도록 수정되었습니다. (Reactor #57)
  • autoloader에서 APPPATH.'third_party' 를 제거하였습니다. 패키지가 존재하지 않거나, 개발자가 기본값으로 어떤 패키지도 로드 하지 않았을경우, 쓸데없는 파일정보를 수집하지 않기 위해서 입니다.
  • (Reactor #231) where 세션라이브러리 예제 SQL이 last_activity의 index를 포함하지 않던 부분을 수정하였습니다. 업그레이드 노트확인.
  • (Reactor #229) 문서에 있는 세션라이브러리 예제 SQL에서 잘못된 부분을 수정하였습니다.
  • (Core #340) $this->db->select() 에서 두번째 파라미터를 전달할때, 서브쿼리에서 칼럼이름이 제대로 escape 처리되지않던 문제가 수정되었습니다.
  • 수정 #199 - 문자열로 전달된 속성값과 여는 태그 사이에 있는 공백은 포함하지 않습니다.
  • Cart Library 의$this->cart->total_items() 함수가 your total count 대신 카트에 있는 모든 아이템의 개수의 합을 리턴하도록 하였습니다.
  • Mysql 과 mysqli 드라이버를 위한 db_forge 에 필드를 추가할때 'null'을 설정하지 않을시, 문서에서 추천하는 대로 NOT NULL 대신 NULL을 기본값으로 하도록 하였습니다.
  • $this->db->select_max(), $this->db->select_min(), 등의 함수가 경고를 throw 할수 있도록 하였습니다.
  • STDIN 에 대한 검사를 php_sapi_name() == 'cli' 로 대신하여 신뢰성을 높이세요. 그러려면, 크론탭 작업에서 파라미터를 가지고 와야합니다.

    • 데이터베이스관리클래스에서 테이블 이름을 변경하지 않은 경우 이름을 생략할 수 있게 되었습니다.
